# 📋 タスク管理拡張システム

既存のOutlook 15分前アラームシステムに追加する、自動タスク管理機能です。

## 🎯 解決する課題
- 前日終業時にタスクメモができない
- 前日に何をしていたか記憶が曖昧
- 朝のルーチン後に何をすべきかわからない
- MSNニュースなどに時間を浪費してしまう

## 🔧 システム構成

### 新規追加ファイル
1. `EndOfDayCapture.ps1` - 終業時自動記録スクリプト
2. `StartOfDayReminder.ps1` - 始業時タスク表示スクリプト
3. `TaskTracker.ps1` - 作業記録管理ライブラリ
4. `TaskSetupExtended.ps1` - 拡張システムセットアップ

### データファイル
- `Data/` フォルダ（自動作成）
  - `daily_tasks.csv` - 日別タスク記録
  - `work_context.json` - 作業コンテキスト
  - `settings.json` - システム設定

## ⚙️ インストール手順

### 1. 拡張システムセットアップ
```powershell
# 管理者権限でPowerShellを開く
.\TaskSetupExtended.ps1
```

### 2. タスクスケジューラ設定（追加）
既存のOutlookReminderタスクに加えて、以下のタスクを追加：

**終業時記録タスク:**
- 名前: `EndOfDayCapture`
- トリガー: 毎日 17:30
- プログラム: `powershell.exe`
- 引数: `-ExecutionPolicy Bypass -File "C:\Scripts\EndOfDayCapture.ps1"`

**始業時表示タスク:**
- 名前: `StartOfDayReminder`
- トリガー: 毎日 09:00
- プログラム: `powershell.exe`
- 引数: `-ExecutionPolicy Bypass -File "C:\Scripts\StartOfDayReminder.ps1"`

## 🚀 動作フロー

### 終業時（17:30）
1. 現在開いているアプリケーション・ファイルを記録
2. 今日の最後のOutlook予定を記録
3. 明日のタスク入力フォームを表示
4. 入力内容をCSVに保存

### 始業時（09:00）
1. 昨日の作業コンテキストを表示
2. 今日の予定一覧を表示
3. ルーチン完了確認
4. 最優先タスクの確認・設定

### 朝のルーチン後
- メール確認完了
- ポータルサイト確認完了
- Teams確認完了
→ 自動で「今日の最優先タスク」を表示

## 📊 記録されるデータ

### 作業コンテキスト
- 開いているファイルパス
- アクティブなアプリケーション
- 最後に編集したドキュメント
- Outlook最終予定

### タスクデータ
- 日付
- 優先度（高/中/低）
- タスク内容
- 完了状態
- 前日からの継続フラグ

## 🎛️ カスタマイズ設定

### 時刻設定
`settings.json`で変更可能：
```json
{
  "endOfDayTime": "17:30",
  "startOfDayTime": "09:00",
  "routineReminderInterval": 30
}
```

### 記録対象アプリ
設定ファイルで監視するアプリケーションを指定可能

## 🔧 トラブルシューティング

### データファイルが作成されない
- PowerShellの実行権限を確認
- Dataフォルダの書き込み権限を確認

### タスクが自動実行されない
- タスクスケジューラの設定を確認
- ユーザーアカウント制御(UAC)の設定確認

### フォームが表示されない
- .NET Frameworkのバージョン確認
- Windows Forms の動作確認

## 📝 使用方法

1. **初回セットアップ後は完全自動**
2. **終業時**: ポップアップフォームに明日のタスクを1-3個入力
3. **始業時**: 表示される情報を確認して作業開始
4. **迷った時**: データフォルダの履歴を確認

## 🔄 既存システムとの連携
- OutlookReminder.ps1はそのまま動作
- 既存のタスクスケジューラ設定も維持
- 新機能は追加のタスクとして独立動作
